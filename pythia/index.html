Pythia

Deze tool maakt alle kernbestanden voor een nieuwe oefening.
Elke stap legt uit waarom het bestand nodig is in Dodona.

Instellingen

Workflow

Werkboom (zonder `preparation/generator.py`)

/example_exercise
|-- config.json
|-- readme.nl.md
|-- description/
|   \\-- description.nl.md
|-- evaluation/
|   \\-- tests.yaml
\\-- solution/
    \\-- solution.nl.py

Stap 1: `config.json`

Waarom nodig: `config.json` maakt van de map een Dodona-leeractiviteit en bevat taal, toegang, titel en evaluatie-instellingen.

Stap 2: `description/description.nl.md`

Waarom nodig: dit is de opdrachttekst die leerlingen zien op Dodona. Markdown wordt ondersteund.

Stap 3: `evaluation/tests.yaml`

Waarom nodig: hierin definieer je testcases (`stdin`, `stdout`, `expression`, `return`) voor TESTed.

Testcase-assistent

`stdin`

De input die naar `input()` gaat. Meerdere regels input? Zet ze op aparte regels.

`stdout`

Exacte uitvoer van de leerlingcode, inclusief hoofdletters, spaties en regelovergangen.

`\\n`

`\\n` betekent een nieuwe regel. Een `print()` eindigt meestal met `\\n`.

Stap 4: `solution/solution.nl.py`

Waarom nodig: voorbeeldoplossingen in `solution/` worden getoond op de oefening-infopagina.

Stap 5: `readme.nl.md`

Waarom nodig: deze readme is extra context voor lesgevers op de info-pagina van de oefening.
